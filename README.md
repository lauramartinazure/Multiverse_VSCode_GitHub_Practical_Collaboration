# Multiverse_VSCode_GitHub_Practical_Collaboration
Repo for Multiverse session on using VSCode and GitHub for practical team collaboration on software projects.

## Session Content:

### The 'why' of using GitHub and VSCode together:
* Why collaboration matters in tech roles
* Why email and/or file versions fail
* What problems Git/GitHub solve (and what's the difference between Git and GitHub)
* Where VSCode fits into the workflow

### VSCode as a collaboration tool as well as an IDE:
* File and Folder Explorer
* Source Control panel
* Extensions
* Integrated terminal (+ how to use it)
* Git indicators
* Timeline/history of a repo
* README.md and markdown as a practice
* .gitignore files

### Practical GitHub workflows in an IDE
* Cloning a repository
* 'Pulling'
* Creating a branch
* Making a change to a single file or multiple
* Committing to a branch and commit messages
* 'Pushing'
* Viewing branches in the GUI and code editor
* Recap of timeline/history of a repo
* Pull requests in the GUI
* Pull requests in VSCode
* Successful merges
* Why you should never commit to main: intentionally creating a conflict
* 'Cherry picking' and dealing with conflicts in code
